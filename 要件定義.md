# AI SaaS アプリケーション 要件定義書

## 1. プロジェクト概要

### 1.1 システム名
AI Image SaaS（AI画像生成・編集サービス）

### 1.2 目的
AIを活用した画像生成・編集ツールをSaaS形式で提供する。フリーミアムモデルを採用し、月額サブスクリプションによるクレジット制で利用料金を管理する。

### 1.3 技術スタック
| カテゴリ | 技術 |
|----------|------|
| フレームワーク | Next.js 14（App Router） |
| スタイリング | Tailwind CSS v4 |
| UIコンポーネント | shadcn/ui（New Yorkスタイル） |
| アニメーション | framer-motion |
| 認証 | Clerk |
| データベース | MySQL（TiDBCloud） / ORM: Prisma |
| 決済 | Stripe |
| AI画像生成 | Stability AI API |
| 画像処理 | sharp |
| デプロイ | Vercel（想定） |

---

## 2. ユーザー要件

### 2.1 ターゲットユーザー
- **クリエイター**: 画像素材を手軽に生成・編集したいデザイナー・イラストレーター
- **マーケター**: SNSやWeb広告向けの画像を素早く作成したい担当者
- **開発者**: 画像処理機能をプロトタイプに組み込みたいエンジニア
- **ビジネスユーザー**: コスト効率よく画像素材を調達したい個人・中小企業

### 2.2 ユーザーストーリー
1. 未登録ユーザーとして、ランディングページで機能・料金を確認し、無料登録できる
2. 登録ユーザーとして、テキストプロンプトからAI画像を生成できる
3. 登録ユーザーとして、アップロードした画像の背景を自動除去できる
4. 登録ユーザーとして、画像を最適化・圧縮できる
5. 登録ユーザーとして、クレジット残高を確認しながらツールを利用できる
6. 登録ユーザーとして、有料プランにアップグレードしてより多くのクレジットを取得できる
7. 有料ユーザーとして、Stripeのビリングポータルでサブスクリプションを管理・解約できる

## 3. 機能要件

| 料金プランセクション | 全プランの比較表（FREE / STARTER / PRO / ENTERPRISE） |
| CTAセクション | 最終的な登録促進バナー |

| ナビゲーション | ロゴ、機能リンク、ログイン/登録ボタン |

### 3.2 認証

| 機能 | 詳細 |
|------|------|
| ユーザー登録 | メールアドレス/パスワードによる新規登録（Clerk） |
| ログイン | メールアドレス/パスワードによるログイン（Clerk） |
| ログアウト | セッション終了 |

| ユーザー同期 | Clerkイベント（作成/更新/削除）をDBに自動反映（Webhook） |

### 3.3 ダッシュボード

| 機能 | 詳細 |
|------|------|
| ホーム | ユーザー名表示、クレジット残高、ツール一覧カード |
| サイドバーナビゲーション | ツール・設定へのリンク（PC）、モバイル対応ドロワー |
| クレジット表示 | 現在のクレジット残高をリアルタイム表示 |
| ツールカード | 各AIツールへのリンクカード（アイコン・説明付き） |

### 3.4 AIツール

#### 3.4.1 画像生成（Image Generator）
| 項目 | 詳細 |
|------|------|
| パス | `/dashboard/tools/image-generator` |
| 入力 | テキストプロンプト（キーワード） |
| 処理 | Stability AI Core Image Generation API呼び出し |
| 出力 | 1280×720 PNG画像（base64エンコード） |
| コスト | 1クレジット/回 |
| 機能 | 画像プレビュー表示、ダウンロードボタン、ローディング表示 |
| エラー処理 | クレジット不足時の警告、APIエラー時のトースト通知 |

#### 3.4.2 背景除去（Background Remover）
| 項目 | 詳細 |
|------|------|
| パス | `/dashboard/tools/remove-bg` |
| 入力 | 画像ファイル（PNG/JPG、最大5MB） |
| 処理 | sharpで前処理 → Stability AI Remove Background API |
| 出力 | 透明背景付き1280×720 PNG画像 |
| コスト | 1クレジット/回 |
| 機能 | ファイルアップロード、処理前後プレビュー、ダウンロード |
| エラー処理 | ファイルサイズ超過、未対応形式、APIエラーの通知 |

#### 3.4.3 画像最適化（Optimize）
| 項目 | 詳細 |
|------|------|
| パス | `/dashboard/tools/optimize` |
| 入力 | 画像ファイル |
| 処理 | sharpによるローカル処理（リサイズ・圧縮） |
| 出力 | 最適化済みPNG画像（品質80%・最大圧縮） |
| コスト | 1クレジット/回（予定） |

### 3.5 サブスクリプション・決済

#### 3.5.1 料金プラン
| プラン | 月額 | 月間クレジット | 主な特典 |
|--------|------|---------------|---------|
| FREE | ¥0 | 5 | 基本機能、コミュニティサポート |
| STARTER | ¥1,000 | 50 | 基本生成、メールサポート |
| PRO | ¥2,000 | 120 | 高度な機能、商用利用可、優先サポート |
| ENTERPRISE | ¥5,000 | 300 | 24/7サポート、APIアクセス、カスタマイズ |

#### 3.5.2 決済フロー
1. ユーザーがプランを選択
2. Stripeチェックアウトセッション作成
3. Stripeホスト決済ページへリダイレクト
4. 決済完了後、ダッシュボードへ戻る
5. Webhookによるサブスクリプション情報の自動更新

#### 3.5.3 サブスクリプション管理
| 機能 | 詳細 |
|------|------|
| ポータル | Stripeビリングポータルへのリダイレクト |
| 確認 | 現在プラン・次回請求日の表示 |
| 変更 | アップグレード/ダウングレード |
| 解約 | Stripeポータルからの解約、FREE状態に戻る |

### 3.6 設定（Settings）

| 機能 | 詳細 |
|------|------|
| プロフィール表示 | 登録メールアドレス表示 |
| プラン表示 | 現在の契約プラン名・次回請求日 |
| サブスクリプション管理 | Stripeポータルへのリンクボタン |

### 3.7 クレジット管理

| 機能 | 詳細 |
|------|------|
| 初期付与 | 新規登録時：5クレジット付与 |
| 消費 | AIツール利用1回につき1クレジット消費 |
| 残高確認 | ダッシュボードおよびツール画面でリアルタイム表示 |
| 不足制御 | クレジット0以下での処理実行を禁止 |
| 月次リセット | サブスクリプション更新時にクレジットをプランに応じてリセット |

## 4. 非機能要件

### 4.1 パフォーマンス
- 画像生成レスポンス：10秒以内（API依存）
- ページ初期読み込み：3秒以内
- Server Actions経由のAPI呼び出し（クレジット不足チェック含む）

### 4.2 セキュリティ
- 全APIルートにClerk認証ガード適用
- ClerkおよびStripe WebhookのSignature検証必須
- サーバーサイドで機密APIキー管理（環境変数）
- Server Actionsによるバックエンド処理の保護
- ファイルアップロード上限：5MB

### 4.3 スケーラビリティ
- TiDBCloud（MySQL互換・サーバーレス）によるDB水平スケール対応
- Prismaのサーバーレスアダプター（@tidbcloud/prisma-adapter）使用
- Next.js App RouterのRSC（React Server Components）活用

### 4.4 ユーザビリティ
- レスポンシブデザイン（PC・タブレット・スマートフォン対応）
- モバイル向けドロワーナビゲーション
- ローディングスピナーによる処理状態の可視化
- Toast通知による成功・エラーフィードバック（Sonner）
- framer-motionによるスムーズなUIアニメーション

## 5. システム構成

### 5.1 アーキテクチャ

```
[クライアント]
  ├── ランディングページ（公開）
  └── ダッシュボード（認証済みユーザー）
        ├── AIツール（Server Actions経由）
        ├── プランページ（Stripe連携）
        └── 設定ページ

[サーバー（Next.js App Router）]
  ├── Server Components（RSC）
  ├── Server Actions（フォーム処理）
  ├── API Routes
  │   ├── /api/generate-image
  │   ├── /api/remove-background
  │   ├── /api/create-portal-session
  │   ├── /api/webhook/clerk
  │   └── /api/webhook/stripe
  └── Middleware（Clerk認証）

[外部サービス]
  ├── Clerk（認証・ユーザー管理）
  ├── Stripe（決済・サブスクリプション）
  ├── Stability AI（画像生成・背景除去）
  └── TiDBCloud（MySQL データベース）
```

### 5.2 データモデル

#### User テーブル
| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| clearkId | String(unique) | Clerk ユーザーID |
| email | String(unique) | メールアドレス |
| stripeCustomerId | String(unique, nullable) | Stripe 顧客ID |
| credits | Int（default:5） | 残クレジット数 |
| subscriptionStatus | Enum | FREE / STATER / PRO / ENTERPRISE |
| createdAt | DateTime | 作成日時 |
| updatedAt | DateTime | 更新日時 |

#### Subscription テーブル
| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| userId | String(unique, FK) | Userへの外部キー |
| stripeSubscriptionId | String(unique) | Stripe サブスクリプションID |
| stripePriceId | String | Stripe 価格ID |
| stripeCurrentPeriodEnd | DateTime | 現在期間終了日 |
| createdAt | DateTime | 作成日時 |
| updatedAt | DateTime | 更新日時 |

### 5.3 Webhook連携

| Webhook | イベント | 処理内容 |
|---------|---------|---------|
| Clerk | user.created | DBにユーザーレコード作成 |
| Clerk | user.updated | ユーザー情報同期 |
| Clerk | user.deleted | ユーザーおよびサブスクリプション削除 |
| Stripe | customer.subscription.created | サブスクリプション作成・クレジット付与 |
| Stripe | customer.subscription.updated | サブスクリプション更新・クレジット更新 |
| Stripe | customer.subscription.deleted | FREE状態に戻す・クレジットリセット |

## 8. 制約・前提条件

- Node.js 18以上
- Stripe・Clerk・Stability AIの各アカウント取得が必要
- TiDBCloudのMySQLデータベースが必要
- Webhookのローカルテストにはngrokなどのトンネリングツールが必要
- 画像アップロード上限：5MB（Next.js Server Actions設定）
- 画像出力サイズ：1280×720（固定）
